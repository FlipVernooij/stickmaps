Stations as base-units

Stickmaps is build around the concept of "survey stations" where a "line" is just the connection between two stations. It is important to remember this when converting from one program to another as others might use lines as their base unit.

Order of measurements

Keeping things clear and making things easy are two different things, in Stickmaps we chose to keep things clear while making them as simple as possible. The most important thing to remember is "Stations are our base-unit", and from there we measure.

Length in
We measure the length from the previous station to the current one.
Azimuth in
We measure azimuth from the previous station to the current one.
Depth
We measure depth of the current station.
Azimuth out
We measure the azimuth from the current station to the next station
Length out
We measure the length from the current station to the next station

We use Drag&Drop

We have chosen for a visual approach when it comes to the user interface. For instance; importing lines to your maps should be a visual experience, just as removing a line should not require you to select stations in a grid-view. As a result of this, some elements in our interface are done by Drag&Drop, there will be no button in order to do so.

SQL enabled

We have chosen to use SQL as a database backend, this enables you as a user to break in to the internal data-structure. If you know (or a re willing to learn) SQL, you can manualy retrieve and/or modify every single piece of data that flows through the application from within any SQL-client of your choice.
